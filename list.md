## 二、多轮测试方案

### 测试环境准备

```
测试前置条件：
1. 服务已启动（bun run dev）
2. TG_BOT_TOKEN 和 TG_CHAT_ID 已配置
3. 至少有 3 个测试任务：
   - 任务A：即将到期（1-3天内）
   - 任务B：已过期
   - 任务C：状态正常（>7天）
```

### 第一轮：后端 API 测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| B-1 | `POST /api/checkin/{taskId}` | 签到成功，返回下次签到日期 |
| B-2 | `GET /api/tasks` | 返回的任务数据包含 `next_checkin`、`days_until`、`status` 字段 |
| B-3 | `GET /api/share/{taskId}` | 返回 shareUrl 和 pageUrl |
| B-4 | `GET /api/share/{taskId}/page?token=xxx` | 显示签到页面，点击签到成功 |
| B-5 | 创建任务后检查 `next_checkin` | 自动计算为 `当前时间 + interval_days` |

### 第二轮：Telegram Bot 命令测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| T-1 | 发送 `/start` | 显示欢迎消息 + 3 个内联按钮（📋 任务列表、⏰ 即将到期、✅ 快速签到） |
| T-2 | 点击"📋 任务列表"按钮 | 显示所有任务列表，每个任务附带签到链接按钮 |
| T-3 | 发送 `/list` | 同 T-2 结果 |
| T-4 | 发送 `/status` | 显示过期和即将到期的任务，每个附带签到链接和到期日期 |
| T-5 | 发送 `/check` | 显示所有任务的签到按钮（点击直接签到） |
| T-6 | 发送 `/due` | 仅显示过期和即将到期的任务，带签到按钮 |
| T-7 | 发送 `/help` | 显示所有可用命令说明 |

### 第三轮：Telegram 内联按钮交互测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| I-1 | `/check` → 点击某个任务按钮 | 该任务签到成功，bot 回复"✅ 任务名 已签到" |
| I-2 | 签到后点击"🔄 刷新列表" | 任务列表更新，已签到任务显示新日期 |
| I-3 | 点击"⬅️ 返回菜单" | 回到主菜单界面 |
| I-4 | `/list` → 点击任务的"🔗 签到"按钮 | 跳转到网页签到页面 |
| I-5 | `/status` → 点击过期任务签到按钮 | 签到成功，状态更新 |
| I-6 | 连续点击同一签到按钮 | 不应重复签到（或提示已签到） |

### 第四轮：网站前端测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| W-1 | 查看任务卡片 | 显示"下次到期"行，包含日期和距今天数 |
| W-2 | 查看过期任务卡片 | "下次到期"显示红色，标注"已过期X天" |
| W-3 | 查看今天到期任务 | StatusBadge 显示橙色"今天到期" |
| W-4 | 签到后刷新 | "下次到期"日期自动更新为 `今天 + interval_days` |
| W-5 | 查看任务列表分组 | 过期/即将到期/正常 三组正确分类 |

### 第五轮：端到端集成测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| E-1 | 网站添加新任务 → TG `/list` 查看 | 新任务出现在 TG 列表中 |
| E-2 | TG `/check` 签到 → 网站刷新 | 网站显示更新后的签到时间 |
| E-3 | 分享链接签到 → TG 收到提醒 | 提醒消息中的日期已更新 |
| E-4 | 等待 cron 触发（9:00/14:00/20:00） | 到期任务收到 TG 提醒，包含具体到期日期 |
| E-5 | 任务过期3天后 | 停止收到过期提醒 |

### 第六轮：边界与异常测试

| 测试项 | 操作 | 预期结果 |
|--------|------|----------|
| X-1 | 无任务时执行 `/list` | 提示"📭 暂无任务" |
| X-2 | 无任务时执行 `/check` | 提示"📭 暂无任务" |
| X-3 | 所有任务正常时执行 `/status` | 提示"✅ 全部正常" |
| X-4 | 所有任务正常时执行 `/due` | 提示"✅ 没有即将到期的任务" |
| X-5 | TG Bot Token 无效 | 服务正常启动，日志提示 TG disabled |
| X-6 | 大量任务（>20个）时 `/list` | 正常显示，不截断 |
